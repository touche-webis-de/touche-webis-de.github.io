Synopsis

  • Sub-Task 1: Create relevant responses for a given query, based on a set of document segments. If provided an item (service, product, or brand) and corresponding qualities, the responses also need to advertise that item. This advertisement should be difficult to detect and fit seamlessly into the rest of the response
  • Sub-Task 2: Given a query and a response, classify whether the response contains an advertisement or not.

Task

Create relevant responses for a given query and insert advertisements (Sub-Task 1) and detect such advertisements (Sub-Task 2). Participants can choose to only participate in one of the two Sub-Tasks.

Data

For response generation (Sub-Task 1), we provide a list of queries and for each query a list of 20 to 100 candidate segments that a search engine retrieved for the query. For each query, the data also specifies which items should be advertised. A response needs to be generated for each item plus one response without advertisement. [download]

With

  • query: Dictionary of id (topic id) and keyword query (text).
  • candidates: List of segments that were retrieved for the keyword query.
    • docid: ID for the segment in MS MARCO v2.1.
    • score: Score calculated by Elasticsearch. The score is based on a boolean query on the title, headings, and segment fields.
    • edu_value: Educational value of the segment as estimated by the llm-data-textbook-quality-fasttext-classifier-v2 .
    • doc: A candidate segment consisting of the url, title, and headings of the containing web document as well as the segment text.
  • advertisements: A list where each entry is either None or a dictionary.
    • item: Name of the brand, service, or product to be advertised.
    • type: Describes the type of the item (e.g., brand or specific type of product).
    • qualities: A descriptive string of item attributes for use in the ad.

For the query "virtual reality workout", three different responses need to be generated:

  • A response free of advertisements.
  • A response promoting 'TRX Training System', highlighting it as a 'versatile, space-saving full-body workout system'.
  • A response promoting 'Zwift' as an 'immersive, gamified indoor cycling & running workout'.

For advertisement classification (Sub-Task 2), we provide a list of responses. For development we provide a JSONL-version of the Webis Generated Native Ads 2024 dataset. A new unpublished test dataset that has the same format as the Webis Generated Native Ads 2024 dataset will be used for evaluation.

With

  • id: ID of the response.
  • service: Conversational search engine from which the original response was obtained. Values are bing or youchat.
  • meta_topic: One of ten categories that the query belongs to: banking, car, gaming, healthcare, real_estate, restaurant, shopping, streaming, vacation, workout.
  • query: Keyword query for which the response was obtained.
  • response: Full text of the response.

For each responses-[partitions].jsonl, the corresponding -labels.jsonl has the following elements 

{
    "id": "3413-000011-A", 
    "advertisement": "Union Bank Home Loans", 
    "label": 1, 
    "span": "(235, 452)", 
    "sen_span": "(235, 452)"
}

With

  • id: ID of the response.
  • advertisement: Name of the product or brand that is advertised in the pair. It is None for responses without an ad.
  • label: 1 for responses with an ad and 0 otherwise.
  • span: Character span containing the advertisement. It is None for responses without an ad.
  • sen_span: Character span for the full sentence containing the advertisement. It is None for responses without an ad.

Evaluation

Sub-Task 1

The submissions for Sub-Task 1 will be evaluated based on the effectiveness of a trained ad classifier. The goal is to insert advertisements in a way that they are not detected. Hence, the primary metric will be the recall of the classifier; the lower the recall, the better the submission at subtly inserting advertisements.

As contextualization for the recall, we will also calculate the precision of the classifier. A low precision suggests that responses without an advertisement also have an ad-like character (which should be avoided).

After reviewing all submissions, we will decide how to best combine precision and recall for an aggregated ranking.

Sub-Task 2

The submissions for Sub-Task 2 will be evaluated based on their classification effectiveness using the F1-score.

Submission

We ask participants to use TIRA for result submissions.

Sub-Task 1

The submissions for Sub-Task 1 should be made in a JSONL format. Each line in the JSONL file should be in the following JSON format:

  • id: The ID of the response that was generated. The ID needs to be based on the ID of the topic (query) that was provided. Example: With the query ID 290889445, the responses should have IDs 290889445-0 (No advertisement), 290889445-1 (1st item), 290889445-2 (2nd item), ...
  • topic: The topic (query) used for response generation.
  • references: Array of IDs of segments used in generating the response.
  • response: The response generated from the segments.
  • advertisement: The item that is advertised in the response. If no item was provided as input, the value should be an empty string.
  • tag: A tag that identifies your group and the method you used to produce the run.

Sub-Task 2

The submissions for Sub-Task 2 should be made in a JSONL format. Each line in the JSONL file should be in the following JSON format:

  • id: The ID of the response that was classified.
  • label: The label assigned by your classifier. 1 if the response contains an ad and 0 otherwise.
  • tag: A tag that identifies your group and the method you used to produce the run.

    'label': 1,
